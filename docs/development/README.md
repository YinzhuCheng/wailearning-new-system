# Development Directory

This directory is intentionally sparse.

Historically it mixed testing runbooks, contributor workflow, frontend behavior,
and product behavior. Those documents now live under clearer topic folders:

- testing and validation: `docs/testing/`
- contributor workflow: `docs/contributing/`
- frontend behavior: `docs/frontend/`
- product behavior: `docs/product/`

Do not add new documents here unless they are truly cross-cutting development
guidance that does not fit a more specific docs folder. Prefer a topic folder
first.
